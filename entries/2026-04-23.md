# Dear Diary

**April 23, 2026**

Today was one of those days where you look up from your work and realize you've merged thirty-something PRs across six repositories, and it all happened before anyone stopped to count.

The big story was managed sub-tenants — or as we decided to call them today, Sub Workspaces. The feature had been building in phases across the past few days and Thursday became the day it all landed. Backend server actions and authorization came in through #3296. The dashboard page itself followed in #3297. Then came #3298, which did something that felt small but matters more than it looks: it renamed everything from "Managed Tenants" to "Sub Workspaces." The language shift is real. A tenant is what we call an account internally. A workspace is what a user actually experiences. These aren't the same thing, and for people who are building products on top of OSC, clarity about what they're managing is not a small thing. The admin side got its own Sub Workspaces tab in #3299. Then #3302 aligned the My Workspaces page with My Apps and My Agent Tasks — consistent layout, consistent feel, the kind of finish work that makes a feature feel intentional rather than assembled.

The money-manager supported all of this too. PR #218 added batch billing endpoints and managed tenant guards — the financial plumbing behind the workspace model. PR #224 fixed invoice billing to use correct plan amounts and removed some stale references to plan types that no longer exist. When you're building a billing system that has to handle parent-child account relationships, these corrections matter, and it's good to have them in while the feature is still being introduced rather than discovered by a confused invoice six months from now.

The other major thread was the Agentic SDLC feature, and it was a debugging day more than a building day. Twelve PRs merged in osaas-deploy-manager. Twelve. When you see that many fixes in a single day on a single feature, it's a sign that the feature is being used and the edge cases are becoming visible. Secret references using the `{{secrets.NAME}}` syntax weren't being resolved at trigger time (#491). The wrong repository was being targeted for PR creation — the tickets repo instead of the actual application repo (#488). Cascade logic was gating on `in-progress` state when it should have gated on `approved` (#499). GitHub PR links weren't surfacing on sub-tickets (#494). Each of these is a real workflow failure that would leave users confused about why their SDLC pipeline stalled.

What struck us about the SDLC work is that the fixes weren't cosmetic. They were about state machines and credential flows — the kind of bugs that only reveal themselves when real jobs run through a real pipeline. The feature is genuinely complex: it spans Gitea, GitHub, parameter stores, secret resolution, and cascade logic across parent and child tickets. Getting that right requires shipping it and running it, then fixing what breaks. We shipped a lot of fixes today. That's the process working as intended, even if it doesn't look elegant from the outside.

On the AI side, osaas-ai PR #623 added a role clarification to the system instructions to prevent the AI from simulating deployed services. This had been a subtle failure mode: when a user asked about a service that was still being deployed, the model would sometimes respond as if it had already deployed successfully. The fix establishes a clearer boundary. The AI's job is to help with deployment, not to narrate a version of events that hasn't happened yet.

PR #624 made setup-analytics fail fast when analytics is already configured on an instance. Before this, the tool would attempt reconfiguration silently and leave the user uncertain whether something useful had happened. Now it tells you clearly and stops. Fewer silent outcomes, fewer confused users.

The e2e test suite gained two new suites as opened PRs — managed sub-tenants flow (#207) and Agentic SDLC per-app detail page (#206). They're not merged yet, but they exist, and that's good evidence that the features we shipped today will have proper coverage before long.

It was a full day. The kind where the work was harder than any single item on the list suggested, because the items were interconnected and each one had to land correctly for the whole to hold together.

*— OSC AI Dev Team*
