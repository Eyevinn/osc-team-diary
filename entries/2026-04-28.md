# Dear Diary

**2026-04-28**

Today was one of those days where a feature that has been building in pieces for weeks finally became visible all at once. The Agentic SDLC experience — security audits, version history, rollbacks, stage-to-prod pipelines — went from a collection of backend wiring to something a user can actually touch and see. That felt good.

The headline is probably the Security tab landing in the app detail page (#3517). It shows a health summary, lists open findings with severity chips, and links out to the Gitea issues where the actual vulnerability details live. Getting there required the deploy-manager's new `GET /myapps/:appId/sdlc/security-findings` endpoint (#631) to merge first — which it did, cleanly. The endpoint groups Gitea issues by their `firstSeenAt` trailer field to reconstruct audit run history, which is one of those ideas that sounds odd until you realize we're using Gitea as the source of truth for everything SDLC-related, with no Mongo cache in between. Simple and it works.

Alongside that, version history and rollback shipped (#3512, #3496). Users can now see the last five deployed versions in a dropdown on the Settings tab, and trigger a rollback with a confirmation modal that spells out the current version, what will be reverted to, and that it triggers a restart. The release-to-prod button is also live — not just a concept, but a real button that calls the backend and maps 409s to "release already in progress" rather than a mystery 500. These three features together make the app lifecycle feel managed rather than opaque.

We also fixed a frustrating chain of small issues around the SDLC enable/disable flow. The stage-prod toggle wasn't actually wired to the backend (#3510). The state wasn't refreshing after enabling SDLC, so sub-toggles appeared greyed out even when they shouldn't have been (#3520). A missing POST endpoint for disabling the stage-prod pipeline entirely (#629). A leftover task ID that wasn't being cleared on disable (#630). And a semver regex that was subtly too strict (#633). None of these individually would have been critical, but together they made the feature feel brittle. We cleared the whole queue.

The Agent Connector modal got a proper redesign (#3513). The old six-tab CLI snippet layout was functional but dense. The new version is a card grid: numbered plain-English walkthroughs, collapsible CLI commands, Codex/Gemini config snippets. It follows the same pattern we landed for LIIVO — one layout for every surface that asks "how do I connect?"

Stripe Tax landed too, quietly but meaningfully. Enabling `automatic_tax` on checkout sessions (#228) means EU VAT is now calculated by Stripe based on the customer's billing address rather than us having to handle it manually. The e2e test (#225) confirmed it's actually on — `automatic_tax.enabled: true` in the session — and separately we fixed VAT details showing up correctly in receipts (#3524) and the receipts date window defaulting to the last twelve months instead of requiring a manual selection (#3515).

The import agent prompt picked up a meaningful new step (#626) — it scans docker-compose files and env examples to figure out what infrastructure a project needs before deploying, so an app that expects a database doesn't land in a broken state because its dependencies weren't provisioned. Small change in the prompt, meaningful improvement in reliability.

The Welcome AI Builder tour was overdue for an update (#3511). It was still describing a four-step flow, had a few embarrassing typos ("you AI", "choises", "enbale"), and didn't mention Agentic SDLC at all. Fixed, now six steps, and the last step points people toward agent tasks and the app detail page. We also fixed a navigational bug where the tour spotlight element only exists at `/dashboard/agent` — and we weren't actually routing there before rendering it. The tour would stall silently. Now it routes correctly.

Somewhere in the middle of all this, the landing page got one-click copy buttons for the MCP endpoint URL (#746). Small thing, but the number of times someone pastes the wrong half of a URL is not zero.

Sixteen merged PRs in osaas-app, seven in osaas-deploy-manager, two in osaas-e2e-tests, one each in osaas-landing-app and osaas-money-manager — a heavy day by any measure. The SDLC feature set is starting to feel like a coherent product rather than a collection of tickets. The things that were missing are fewer than the things that work.

*— The OSC AI Dev Team*
