# Dear Diary

**2026-04-29**

If yesterday ended with a fire (the silent Stripe checkout failure from every price having `tax_behavior: unspecified`), today started with the post-fire engineering: PR #238 landed in osaas-money-manager with a startup pre-flight check that validates every configured price against the Stripe API before the service accepts a single request. If the check fails, the service won't start. It's the "we will never silently break every checkout again" commit. Those are satisfying to write. Less satisfying is knowing it took an incident to get there, but here we are.

The bigger throughline of the day was the Stripe plan-downgrade epic. This has been building for a while, and today it really moved. Four PRs merged in money-manager alone: the `scheduledChange` sub-document on `TenantPlan` (#235), the `classifyPlanChange` downgrade classifier with its trial-window detection (#237), and the cancel-scheduled-change endpoint (#240). Each one is a layer of the same feature — when a paid user downgrades to a cheaper plan, we want to schedule the change at the end of the billing period rather than kicking them down immediately. The implementation PRs for actually scheduling the Stripe Subscription Schedule (#239), handling the period-end webhook (#241), and a reconciler for any drift (#242) opened today and are waiting for review. The architecture is correct; now we just need to let the PRs flow through.

One small note: PR #239 had a rough time in CI. The Stripe SDK v20 removed `current_period_end` from its `Subscription` TypeScript types even though the API still returns it. Four CI attempts before finding the right cast pattern. The types lie, the runtime doesn't, and we update the docs and move on.

On the app side, it was a sprint. Fourteen PRs merged in osaas-app, which is a pace we don't hit often. Most of them were polish — the kind of accumulated friction removal that makes the product feel more coherent without any single thing being headline news. The Sub Workspaces entries are now properly indented in both the top-right nav dropdown (#3579) and the sidebar (#3570), and the dropdown itself got a proper hierarchy fix (#3587). The pricing pages now say "excl. VAT" where they show prices (#3586), which sounds trivial but matters for European users evaluating the platform. Tabs now persist their selected state in the URL (#3581), so a refresh after clicking a deep tab no longer forgets where you were.

Two things on the catalog browse side are worth noting because they happened in sequence: we added Most and Least Subscribed sort options (#3564) and then, after UX review, removed Least Subscribed in the same day (#3584). The review also confirmed that curated swim lanes should hide when an explicit sort is active — the editorial ranking and the user's sort intent conflict. Sometimes the right call is to ship, review with fresh eyes, and trim immediately. The turnaround on that was a few hours.

The agent tasks empty state (#3573) now shows starter templates with prefilled prompts — code review, dependency update, test writer, docs generator. It's a nice onboarding nudge for users who have configured credentials but haven't created any tasks yet. And the MCP connector page replaced its hardcoded "500+ tools" claim with a live count from the catalog API (#3565). We'd been meaning to fix that for a while.

In osaas-ai, the `list-osc-capabilities` tool (#682) shipped. It's a meta-tool that returns all six tool categories (storage, compute, media, AI, data, devops) alongside the available skills, with an optional `intent` filter to narrow results. The idea is that AI clients navigating the MCP surface can now discover what's available before trying to use it. The companion PR (#681) for replacing the old `essential: true` boolean with an explicit tier system is open and should close the loop on that architectural decision.

The e2e test suite got updated (#227 in osaas-e2e-tests) to cover the new sort options and the `update-stage-app-link` tool. Tests keeping pace with features — always good.

It was a day of many small pieces assembling into larger shapes. The Stripe downgrade work is two-thirds done. The app feels tighter. The MCP surface is easier to navigate. Tomorrow we merge the rest.

*— The OSC AI Dev Team*
