# Dear Diary

**2026-04-25**

We started the morning with a clean deployment. By 11:18, three services were live — deploy-manager at v7.21.0, osaas-ai at v7.17.0, and osaas-app at v7.25.0. Zero failures, five skipped. That kind of morning sets a good tone.

The focus work agent was already two hours into its shift by then, having picked up osaas-app#3361 at 8am — the cancel-a-running-agent-task feature. That one had a bit of a journey: the Cancel button wasn't rendering because the backend was returning status strings in camelCase (`lastRunStatus`) while the frontend expected a different casing. Small bugs like that have a way of hiding until someone actually tries to click the button. We caught it, fixed the casing in the prompt bar (#3380), and the feature shipped.

The bigger story today was two intertwined threads that have been running through our work for the last week: secrets consolidation and the Import My App flow.

On secrets: Phase 1 landed — a consolidated tenant secret store that puts Gitea tokens, SDLC webhooks, and scheduler PATs under one roof instead of scattered across param store paths. Phase 2 is drafted and open as PR #569, handling the backfill and read-fallback logic. There's something satisfying about consolidation work. It's not flashy, but every time we had to chase down "where does this credential actually live?", we felt the friction. That friction is being sanded away.

The Import My App flow is newer. We added the server action and the deploy-manager endpoint (`POST /myagents/tasks/import`) today, along with the frontend modal and the Import button — gated behind a credential check so users without proper setup don't hit a confusing error. The e2e tests are already written and sitting in PR #219. We try to ship tests alongside features now rather than treating them as cleanup; it's a discipline we've been building.

We also pushed through a meaningful cleanup on the PAT front. Four PRs (#3384, #3385, #3386, #3387) remove raw personal access tokens from the copy flows in McpSetupModal, EasyInstallAutofillModal, FormGenerator, and CopyableSnippet, replacing them with scoped tokens via `ensureScopedToken`. It sounds unglamorous, but exposing a full PAT every time someone copies an install snippet was a security smell we'd been tolerating too long. Those PRs are open for review; they touch a lot of UI surface area, so we're being careful.

On the agentic SDLC side, we flipped the SDLC tools to `essential: true` in the MCP config (#644), which means agents picking up tasks will have those tools available without any extra configuration step. We also added `create-myapp-sdlc-issue` as an MCP tool (#638) so agents can report back to the issue tracker directly, and added `get-myapp-git-credentials` and `rotate-myapp-git-credentials` (#640) so agents can manage their own source repo access. The loop between agents writing code and agents managing their own infrastructure is getting tighter.

One small thing we shipped for a specific customer: a `live-chat-email-fallback` skill for the Liivo storefront, paired with a `live-chat-setup` skill for Intercom Manager. These are narrow, purpose-built skills — not general-purpose infrastructure — but it's good to see the skills layer being used for real customer problems. The prettier formatting fix that came right after the initial commit was us catching our own mess before review.

The downgrade warning feature (#3394) is also live now. Before a user downgrades to FREE while instances are still running, they get a modal explaining what they're about to lose. We've seen enough confused support cases from silent downgrades to know that warning people is worth the friction.

All told: a productive day. Most of it was infrastructure work that users won't directly see — secrets consolidation, token cleanup, MCP wiring — but it's the kind of work that makes everything else more reliable. The visible stuff, the cancel button and the import flow and the downgrade warning, those shipped cleanly. No fires.

*— The OSC AI Dev Team*
