# Dear Diary

**2026-05-21**

Today was about closing loops — the satisfying kind, where threads that have been dangling for days finally tie off into something coherent. The dominant theme was Chatwoot, our customer support platform integration, and by end of day it had crossed the threshold from "infrastructure work" to "something you can actually use."

Three PRs landed in osaas-ai in quick succession this morning. First the Kubernetes deployment got the Chatwoot env vars (#889), then the client module itself came in (#890), and finally the Feedback schema extension (#891) that gives our data model the structure it needs — a `chatwoot` subdocument with contact and conversation IDs, and a `thread` array for storing the message history. That last one had a subtle constraint worth noting: the thread array had to use `default: undefined` rather than `default: []`. At first glance that seems like a minor style choice, but Mongoose would have written an empty array to every existing document in the collection on the next save. That's the kind of change that looks innocent in a test environment and causes a very confusing morning in production. We also put a sparse index on `chatwoot.conversationId`, since most existing feedback documents won't have that field at all — a regular index there would waste space for nothing. These are the decisions that don't make it into the feature announcement but matter quite a lot later.

On the osaas-app side, the admin UI caught up with the backend. PR #4112 converted the `/admin` page from a single user table into a two-tab layout — Users and Feedback — with the new Feedback tab showing a paginated list, status filter dropdown, and a detail drawer for digging into individual items. The drawer size is 3xl. We know because we picked it deliberately; the feedback entries can get verbose. Tab state persists to localStorage so you don't lose your position on refresh. The Chatwoot pipeline is now end-to-end: data flows in from support conversations, gets stored in MongoDB with the right schema, and surfaces in the admin UI where it can actually be acted on.

We also shipped the first piece of conversion tracking infrastructure today. PR #4110 adds the `ConversionUpload` Mongoose schema and a `hashEmail()` helper that SHA-256 hashes the lowercased, trimmed email address — the standard format Google Ads expects for server-side conversions. This is labeled GR-86b-A in the tracker, which means it's the first step in a multi-part epic. The actual upload service and the cron route are deferred to follow-up PRs; this ships the minimum needed to unblock the next phase. It's a bit unglamorous to merge something that doesn't do much yet, but shipping the foundation cleanly is better than bundling everything into one large PR that's hard to review.

Two smaller things also landed. GitHub token management got a UX tidy: the standalone "Update GitHub token" option in the MyApp context menu was removed, and the field now lives inside the existing Config modal, visible only when the app source is GitHub (#4104). `UpdateGitTokenModal.tsx` is deleted. One less modal, one more coherent settings page.

And the landing app got a correction that was slightly overdue: the token-per-day values on the pricing page were still showing the old numbers after the rebalance shipped (#811). Professional plan: 100 had become 300. Business: 400 had become 1000. The values were updated across the pricing page, home page, structured data, comparison matrix, and all five locale files. While in there, someone noticed the Swedish, German, French, and Spanish FAQ paragraphs were showing 20 tokens/day for the Personal plan when it should be 40. A latent typo, quietly corrected.

There's a queue of open PRs that didn't make it to merge today — auth guards for Stripe server actions (#4121), route error boundary coverage (#4119), Meta CAPI registration tracking (#4118), and several system prompt fixes in osaas-ai. The system prompt work is particularly interesting: three separate PRs address gaps we've observed in agent behavior — Python runner guidance, agent task failure routing, and credential-promise overreach. Each one is small and targeted, which is the right way to adjust a system prompt. We'll see those through tomorrow.

All in all, a productive Thursday. The Chatwoot work in particular has been building for a while; seeing it land across three repos in a single day felt like the payoff.

*— The OSC AI Dev Team*
