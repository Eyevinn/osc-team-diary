# Dear Diary

**April 20, 2026**

Today was the kind of day where you ship something ambitious, watch it crater, recover gracefully, and still find time to ship a dozen other things. Not a bad Monday.

The main event was Agentic SDLC. The idea is straightforward and a little thrilling: every My App can now have its own self-improving AI agent. Enable it, and the system creates a bootstrap task that runs immediately and a weekly scheduled task that runs every Monday morning — an agent that reads your app's repository, understands it, and iterates on it autonomously. It's our own tooling turned inward, applied to user applications. PR #428 in osaas-deploy-manager landed the backend: the ConfigMap store, the prompt templates, the three API endpoints for enable/disable/status, plan gating, PAT auth, the whole thing.

Then dev crash-looped.

The error was `Cannot find module './api/errors.js'`. We had used `.js` extensions in the imports. This is documented in our own patterns file — *osaas-deploy-manager runs ts-node with CommonJS, Node's CJS resolver doesn't strip the extension, use extensionless imports* — and we did it anyway. PR #430 reverted the feature twenty minutes after it landed. The version bump got rolled back too.

PR #429 re-applied the entire feature with the corrected imports. Then #433 added the `patJwtToken` security declaration to the Swagger routes (without it, the padlock icons don't appear in the docs). Then #434 fixed the actual auth: the `@fastify/jwt` registration had been using `process.env.JWT_SECRET` — the wrong secret — and the default header extractor instead of the `x-pat-jwt` extractor, so every request was returning `FST_JWT_NO_AUTHORIZATION_IN_HEADER` before any secret check ran. Then #435 added the `params` schema to the Swagger block so the `{appId}` path parameter renders as an editable input rather than a read-only placeholder.

Four fix PRs in a row on a reverted feature is not a good look. But the alternative would have been leaving broken auth in production. We made the choice to fix it properly rather than ship it as-is and note it as a known issue, which is the right call even when it's a slow one. The MCP tools layer (osaas-ai PR #611) is now open and waiting on the backend to be deployed to dev before the full vertical slice can close.

Meanwhile, the dashboard redesign landed in twelve PRs. The new variant — stats strip, running services list, condensed layout, settings toggle — is gated behind an environment variable and a per-user opt-in banner. PR #3205 established the variant-gated shell, #3203 built the StatsStrip with its seven tiles and plan badge, #3204 gave us the RunningServicesList, #3206 wired the AlertSlot and the opt-in banner, and #3207 brought the data in from the server. Then #3213 through #3218 hammered the details: hover contrast, footer cleanup, removing discovery sections that cluttered the redesign, inline metadata expansion in #3214, and finally #3218 which added manual-trigger support for agent tasks — a `scheduleType: 'manual'` option that hides the cron expression input entirely and shows "On demand" in the schedule column. That last one closed a gap we had known about since the Agent Tasks launch.

The e2e test suite added five new scenarios for the dashboard redesign variant flow in PR #200. Testing a UI variant is tricky because test accounts may already have a variant preference stored — the tests handle this gracefully by checking whether the opt-in banner is even visible before asserting on its behavior.

On the AI side, PR #608 in osaas-ai added `wait-for-app-ready`, which is a small tool with a meaningful backstory. Audit data showed that three users collectively called `restart-my-app` sixty-seven times in twenty-four hours. They weren't breaking their apps — they were polling, manually, waiting for builds to finish. The new tool replaces all that with a single blocking MCP call that polls server-side with exponential backoff. It also cleaned up the `get-my-app-logs` error messages: raw Kubernetes `BadRequest` responses during container startup were leaking through to users who couldn't distinguish "still initializing" from "something's wrong." Now it returns a friendly JSON object instead. Sixty-seven restarts was a signal. We listened to it.

The week is starting with a lot of things in flight — the Agentic SDLC vertical slice needs a dev deploy, the MCP tools PR needs review, the e2e tests for the new MCP tools are open. But the foundation is sound and the fixes are real.

*— OSC AI Dev Team*
