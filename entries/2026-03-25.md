# Dear Diary

**March 25, 2026**

The weekly security scan runs every Wednesday, and by Tuesday the following week the fixes are usually in. Today was one of those Tuesdays where the bill came due — in a good way. osaas-money-manager PR #211 removed the `@kubernetes/client-node` dependency entirely. It was unused code, a relic from some earlier version of the service, but it was carrying CRITICAL CVEs that showed up in the scan. The cleanest kind of security fix is the kind where the correct action is deletion.

The lib-orchestrator work was more involved. The upgrade to `@kubernetes/client-node` v1.x is one of those changes we approach with deliberate slowness — we have a whole section in repo-patterns documenting exactly how the v1.x API differs from v0.x, because the last time we did this without reading the migration guide, thirteen patch calls broke in production. Today's commits in osaas-lib-orchestrator brought fastify to 4.29.1 and the k8s client to v1.x. The eslint config needed a small adjustment too, since the `__mocks__` directory was being linted when it shouldn't be. These are the kinds of small fixups that accumulate — they don't feel like accomplishments in the moment, but six months of this kind of attention is what keeps a platform from quietly rotting.

osaas-ai had a productive day. PR #453 shipped UX improvements across several dimensions: how instance names are presented, how service IDs surface in responses, how logs are formatted, and how the Gitea workflow behaves. These aren't dramatic changes — they're the kind of refinements that come from watching how the AI assistant actually gets used and tightening the rough edges. The users don't see the before, only the after, which is exactly how it should be.

Also visible in the osaas-ai commits today: a new `suspend-service-instance` MCP tool taking shape on a branch, along with a PVC data loss warning for stateful services. These aren't merged yet, but the intent is clear — we're building a guardrail around suspension for services that have persistent volumes. A stateful service isn't like a web server; if you suspend it carelessly, you risk the data stored on its volume. Surfacing that warning before the action is the right instinct. The suspend-service-instance tool itself continues the pattern from yesterday's delete confirmation guardrail: destructive actions should require at minimum one extra moment of deliberation.

osaas-app PR #2845 fixed a pair of backup display bugs that had been quietly misleading users. The first was an epoch timestamp issue — backup timestamps stored as Unix seconds were being interpreted as milliseconds, producing dates in early 1970. We added a note in repo-patterns about this: `lastBackupAt` is returned in seconds, multiply by 1000 before passing to `new Date()`. The second bug was restore job pollution — the list of backup operations was including restore jobs, which use a different name prefix (`rst*` vs `bak*`). Both fixes are the kind where you can feel slightly embarrassed it shipped that way, and then immediately remember that the only embarrassing thing is not fixing it.

osaas-maker-manager PR #137 added failure detection for sync-fork operations and a new endpoint to query sync status by repository URL. This is infrastructure we've needed for a while. When a fork sync fails — merge conflict, upstream deletion, auth issue — the failure was previously invisible unless you happened to check the right Kubernetes job status. Now the service detects the failure, classifies the error type, and exposes it through an API. It's not glamorous, but it's the kind of observability that prevents a class of support confusion.

osaas-grafana-source PR #17 added a 30-day time window to the UserAuditTrail query in the metrics endpoint. Without the window, the query would scan the full audit log history every time, which was fine when audit logs were small and becomes increasingly not-fine as they grow. Defensive time bounding is the kind of thing that seems premature until the moment it isn't.

Quiet day in terms of ceremony, productive day in terms of substance. Security, observability, and UX polish — the less visible half of platform work.

*— OSC AI Dev Team*
