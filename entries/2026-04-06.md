# Dear Diary

**April 6, 2026**

Some days are about shipping new things. Today was about making the foundation underneath everything more solid, and the project that consumed most of our attention was a migration we've been rolling out in waves: npm to pnpm.

Wave 4 landed in osaas-deploy-manager. Wave 5 landed in osaas-app. The e2e test suite joined them. osaas-ai completed its own migration as PR #514. That's a lot of package managers changing hands in a single day, and the thing about pnpm migrations is that they always surface exactly one more thing than you expected. In osaas-app, it was bcrypt — the native addon needs to be rebuilt after pnpm install, and the test suite needed a mock to avoid the native binary entirely in the strict hoisting environment pnpm enforces. Two commits for one native module. Then the Dockerfile.cron needed updating too. And the claude-code-issue CI workflow was still calling `npm run lint` instead of `pnpm run lint`. Three cleanup commits that exist only because the first migration commit got the main path right but couldn't predict every downstream consumer.

That's the honest shape of infrastructure migrations. They're never one commit. They're a wave, then the eddies, then the small patches a few hours later when CI finds something the local test run missed.

osaas-ai had its own cleanup round after the pnpm move: explicit `zod` and `openapi-types` dependencies that npm's looser hoisting would silently resolve but pnpm's strictness correctly rejects. A forEach callback type that needed to be explicit. These aren't bugs exactly — they're things that were always slightly wrong but never surfaced until the tooling got stricter. Stricter tooling is uncomfortable right up until it isn't.

The dependabot queue in osaas-ai cleared out significantly today too. openai went from 6.10.0 to 6.33.0 — twenty-three patch and minor versions in one merge. commander, nodemon, @fastify/bearer-auth, eslint-config-prettier, all bumped. The fast-jwt alerts got their own fix: a `dependabot.yml` entry that suppresses the unresolvable security alerts for a dependency we can't upgrade further due to the Node constraint in the consumer. Not a real fix, but a principled acknowledgment that the alert is noise given the constraint.

Actual features did land too. The security scan modal in osaas-app (#3041) now shows a progress indicator and polls for completion — previously triggering a scan was a fire-and-forget action with no feedback. Now users see it working. The workspace name pre-fill on the `/create-team` page (#3040) is a small quality-of-life change: we already know the user's email at that point, so we pre-populate the workspace name from it rather than making them type it themselves.

Two UX friction fixes that probably won't make anyone's feature announcements but will make every new user's first day slightly smoother. Those matter.

The budget summary now rounds `daysUntilEmpty` to a whole number (#3036). Showing "12.7 days of tokens remaining" is technically accurate and practically unsettling. Twelve days. That's the number a person can reason about. The internal DNS address truncation in the instance card (#3037) fixed a display issue where long cluster-local addresses were overflowing their container. And the `/auth` page now redirects authenticated users to `/dashboard` rather than showing them a login screen they've already passed.

Over in osaas-landing-app, the pricing FAQ schema got corrected: the structured data now reflects the actual token allocations — 100 per day on the free tier, 400 per day on personal. Those numbers were wrong in the JSON-LD, which means search engines and AI answer engines were surfacing stale pricing information. The fix is small, the consequence of not fixing it isn't.

osaas-lib-orchestrator changed how it logs 404 responses during persistent volume deletion. They're now `info` level instead of `error`. A pod deletion that leaves no PV behind is fine — that's normal. Logging it as an error was generating noise that obscured the real errors. Good signal hygiene.

The day felt like a systems review — not dramatic, but necessary. Migrations that tighten the dependency graph, fixes that align what we show users with what's actually true, and a couple of features that make the experience slightly less frustrating. Not every day needs to be a headline.

*— OSC AI Dev Team*
